# OWASP CRS Rule `930100` — Path Traversal (`../` Attacks)

> **Scope of this file:** the single canonical reference for OWASP CRS rule `930100` — detection of relative path-traversal sequences (`../`, `..\\`, URL-encoded `%2e%2e/`, double-encoded `%252e%252e/`, overlong UTF-8 `..%c0%af`) — and the false positives it produces on import-map URLs, source-map URLs, JavaScript ES-module specifiers, dependency-lock manifests, CDN paths, and any legitimate use of relative paths in request inputs. Covers Google Cloud Armor, AWS WAF, and Cloudflare WAF side by side.
>
> Related rule `930110` ("Path Traversal Attack (`/../`)") catches the same family but with the leading slash explicitly required — most field-scoped exclusions for `930100` need to cover both. See §2.4.

---

## 1. The Symptom

| Cross-provider signature                                                                                                                                    | Where you see it            |
| ----------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------- |
| `HTTP 403` on GETs that legitimately contain `../` in the URL path or query string                                                                          | Browser dev tools → Network |
| ES-module / import-map / source-map loading fails — browser console shows `403 Forbidden` for resources like `/assets/js/foo.js?map=../foo.js.map`          | DevTools console + Network  |
| Single-page-app routes that encode relative paths as part of the URL (e.g. an in-app file-browser whose URL is `/files?path=../docs/intro.md`) return `403` | SPA in-browser navigation   |
| Webhook payloads / API requests with body fields containing relative-path strings (`{"include": "../config/base.yaml"}`) refuse to save                     | Config-update API           |
| GraphQL / OData queries containing `..` as a navigation operator (rare but real) get blocked                                                                | API gateway                 |
| Build / CI proxies pulling artifacts from a registry whose URL contains `..` (e.g. NPM-style scoped paths) fail                                             | CI logs                     |
| Generic edge block page; application logs empty for the failing request                                                                                     | App access log              |

Distinguishing fingerprint: the failing payload contains `..` as a path segment (with or without URL encoding), and the input is **a URL or path string**, not arbitrary text containing `..` (e.g. `please proceed..` in a comment body does **not** typically trip `930100` because of the boundary requirements — see §2.1).

## 2. The Diagnosis

### 2.1 What rule `930100` does

Rule `930100` is a regex rule. Its modsecurity-style signature:

```
SecRule REQUEST_URI_RAW|REQUEST_BODY|REQUEST_HEADERS|XML:/*|!REQUEST_HEADERS:Referer|ARGS|ARGS_NAMES
  "@rx (?i)(?:(?:^|[\\/])\.{2,3}[\\/]|[\\/]\.{2,3}(?:[\\/]|$)|(?:^|[\\/])\.{2,3}$)"
  "id:930100,
   phase:2,
   block,
   capture,
   t:none,t:utf8toUnicode,t:urlDecodeUni,t:removeNulls,t:cmdLine,t:normalizePath,
   msg:'Path Traversal Attack (/../) or (/.../)',
   ..."
```

Key facts:

- The match operator is **`@rx`**, case-insensitive.
- The pattern matches **two or three consecutive dots** (`..` or `...`) **bracketed by path separators or start/end of input**. The leading dot-count `\.{2,3}` is what catches the rare "triple-dot" filesystem traversal trick (`...` on some legacy Unix shells).
- The transformation chain `urlDecodeUni → utf8toUnicode → removeNulls → cmdLine → normalizePath` is **the entire reason this rule is hard to evade and hard to predict**:
  - `urlDecodeUni` decodes `%2e%2e%2f` → `../`.
  - `utf8toUnicode` collapses overlong UTF-8 encodings (`%c0%ae` → `.`, the classic IIS unicode bug).
  - `normalizePath` resolves embedded path operations: `foo/./bar` → `foo/bar`, `foo//bar` → `foo/bar`, and importantly `foo/../bar` → `bar`.
  - The order is significant: `normalizePath` runs **after** decoding, so encoded traversal sequences become canonical before regex match.
- Default targets are extremely broad: `REQUEST_URI_RAW`, `REQUEST_BODY`, `REQUEST_HEADERS` (except `Referer`), XML element values, `ARGS`, `ARGS_NAMES`. **Notable: the URI path itself is inspected**, not just decoded `ARGS`.

### 2.2 The exact legitimate inputs that trip it

| Submitted shape                                                                                                                                                                                                                  | Why the rule fires                                                                   |
| -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------ |
| Source-map URL: `/assets/js/main.js?map=../main.js.map`                                                                                                                                                                          | `?map=` value contains `../`.                                                        |
| ES-module specifier passed through a query string: `/loader?mod=../shared/utils.js`                                                                                                                                              | Same.                                                                                |
| Import-map JSON body field: `{"imports": {"foo": "../vendor/foo.js"}}`                                                                                                                                                           | `ARGS` body field value matches.                                                     |
| In-app file-browser route: `GET /files/api?path=../docs/intro.md`                                                                                                                                                                | Query value matches; URI also matches if the SPA encodes the path in the URI itself. |
| Package-manager scoped path used as proxy URL: `/registry/@scope/pkg/-/pkg-1.0.0.tgz?dir=../alt`                                                                                                                                 | Same.                                                                                |
| Webhook body from a build tool: `{"diff": "--- a/foo.txt\\n+++ b/../bar.txt\\n"}` (unified-diff hunk header with `../`)                                                                                                          | `REQUEST_BODY` regex match on `/../`.                                                |
| Markdown content with a literal relative link: `[See](../docs/index.html)` — submitted to a content API                                                                                                                          | Body match.                                                                          |
| OData / GraphQL relative navigation operators: `$select=../parent/id`                                                                                                                                                            | Body or query match.                                                                 |
| CDN path used as a forwarding rule: `/proxy?upstream=https://cdn.example.com/v1/../v2/asset.js`                                                                                                                                  | Query value match.                                                                   |
| Browser preloading: `<link rel="modulepreload" href="../shared.js">` triggering an HTTP request for `../shared.js` resolved by the browser to a real path, but reported in `Referer` or other headers depending on the SPA shell | Header match (depending on which headers the WAF inspects).                          |

The single most common production cluster: **source maps and ES-module loaders**. Modern build tools emit URLs that intentionally encode relative paths, and they fire `930100` on the **first browser load** after a deploy.

### 2.3 What the rule does **not** catch (and why that matters)

The rule does **not** match on:

- `..` as a substring inside a word that does not have path-separator boundaries (e.g. `please proceed..` in a chat message — no slash before or after).
- A single dot (`./`) — that's covered by `930120` (OS file access attempt) only when followed by sensitive system filenames.
- A `..` segment that `t:normalizePath` resolves away **inside the same string** (e.g. `foo/bar/../baz` may be normalized to `foo/baz` before regex evaluation, depending on the exact normalizer behavior — CRS implementations vary).

This is important because **the offline regex check (§3.5) can give false negatives** if you don't apply the transformations in the same order the WAF does. Always reproduce against the live WAF in preview mode before declaring "but the regex doesn't match".

### 2.4 Sibling rules that co-fire

| Rule     | Detector                                                                                  | Why it co-fires                                                                         |
| -------- | ----------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------- |
| `930100` | Path traversal `(/../)`, `(/.../)`                                                        | The primary terminator.                                                                 |
| `930110` | Path traversal — leading-slash variant                                                    | Often duplicates `930100` matches; usually both fire on the same payload. Exclude both. |
| `930120` | OS file access attempt — `/etc/passwd`, `/proc/`, `/sys/`, `c:\windows\` style references | Independent but adjacent; fires on payloads that reference well-known OS paths.         |
| `930130` | Restricted file extensions — `.bak`, `.swp`, `.git`, `.svn`, `.env`                       | Co-fires when traversal targets a sensitive extension.                                  |
| `932160` | RFI — `file://`, `php://`                                                                 | Independent but in the same RCE-adjacent neighborhood.                                  |

Always exclude `930100` **and** `930110` together when tuning legitimate traversal-shaped inputs; they are functionally redundant on most modern engines.

### 3.5 Offline regex sanity check

```bash
python - <<'PY'
import re, urllib.parse, os.path

CRS_930100 = re.compile(
    r"(?i)(?:(?:^|[\\/])\.{2,3}[\\/]|[\\/]\.{2,3}(?:[\\/]|$)|(?:^|[\\/])\.{2,3}$)"
)

def transform(s):
    s = urllib.parse.unquote(urllib.parse.unquote(s))   # urlDecodeUni (approx, double-decode)
    s = s.replace("\x00", "")                            # removeNulls
    s = os.path.normpath(s).replace("\\", "/")           # normalizePath (approx)
    return s

samples = [
    "/assets/js/main.js?map=../main.js.map",
    "/assets/js/main.js?map=%2E%2E%2Fmain.js.map",
    '{"imports":{"foo":"../vendor/foo.js"}}',
    "please proceed..",                  # benign
    "do you know if it's the cat..?",    # benign
]
for s in samples:
    t = transform(s)
    m = CRS_930100.search(t)
    print(f"{s!r:60s} -> normalized={t!r:50s} match={bool(m)}")
PY
```

(The `t:normalizePath` step in WAF engines is sometimes more aggressive than `os.path.normpath`; consider the offline check a lower bound — if it matches offline, it definitely matches on the WAF; if it doesn't match offline, the WAF may still match it.)

## 4. The Remediation Matrix

> **House rule:** never disable `930100` globally. Scope by **path** and, where possible, by **specific parameter or asset prefix**. Roll out in **preview / count / log** mode for 72h. Pair with `930110` in the same exclusion.
>
> **Architectural fix preferred over WAF exclusion for source maps.** The right fix for source-map FPs is to publish source maps with absolute paths (or under a stable `/sourcemaps/` prefix that doesn't require traversal), and remove the `?map=../…` query pattern from the build tool's output. Track that work in a JIRA / GitHub issue and link it from the audit row.

### 4.1 GCP Cloud Armor

Field-scoped exclusion for source-map style asset paths:

```hcl
rule {
  action   = "allow"
  priority = 920
  preview  = true

  match {
    expr {
      expression = "request.path.startsWith('/assets/')"
    }
  }

  preconfigured_waf_config {
    exclusion {
      target_rule_set = "lfi-stable"
      target_rule_ids = [
        "owasp-crs-v030301-id930100-lfi",
        "owasp-crs-v030301-id930110-lfi",
      ]

      request_query_param {
        operator = "EQUALS"
        value    = "map"
      }
    }
  }

  description = "Exclude ?map= query param from 930100/110 on /assets/*. Build-tool emits relative source-map URLs (JIRA-5678 to switch to absolute). EXCEPTIONS.md row YYYY-MM-DD."
}
```

For SPA file-browser routes that need to carry traversal-shaped paths in a `path` query param:

```hcl
rule {
  action   = "allow"
  priority = 925
  preview  = true

  match {
    expr {
      expression = "request.path.matches('/files/api')"
    }
  }

  preconfigured_waf_config {
    exclusion {
      target_rule_set = "lfi-stable"
      target_rule_ids = [
        "owasp-crs-v030301-id930100-lfi",
        "owasp-crs-v030301-id930110-lfi",
      ]

      request_query_param {
        operator = "EQUALS"
        value    = "path"
      }
    }
  }

  description = "Exclude ?path= query param from 930100/110 on /files/api. Backend rejects requests that resolve outside the user's home dir. EXCEPTIONS.md row YYYY-MM-DD."
}
```

> **Backend-side check is non-negotiable.** Once the WAF stops checking the path string, the backend MUST canonicalize the requested path and reject anything that resolves outside the user's allowed root. Document the relevant code path / commit SHA in the audit row's justification column.

## 6. Common pitfalls

- **Excluding only `930100` and watching `930110` block next.** Always exclude both together.
- **Transformation order matters.** Decoded + normalized payloads can match the regex even when the raw payload doesn't, and vice versa. Reproduce against the live WAF in preview mode before concluding.
- **`Referer` is excluded from the rule's target set by default.** A `Referer` header containing `../` will NOT trigger `930100` even though it would trigger other rules. Useful to know when debugging "but the URL definitely had `../` in it — why didn't the WAF block?".
- **AWS body-inspection size limit.** Long JSON config payloads can have their `../` content past the cutoff. Document the configured limit in the audit row.
- **Backend canonicalization is the actual security boundary.** Once the WAF stops inspecting the `path` param, the backend must canonicalize and reject out-of-root resolutions. The audit row must reference the canonicalization commit.
- **Cloudflare double-coverage.** OWASP CRS override does not address the Cloudflare Managed Ruleset's own LFI heuristics; check `terminating_rule.id` after promoting.
- **`%2e%2e%2f` vs `..%2f` vs `..%5c` vs `%c0%ae%c0%ae%2f`.** The decoder collapses all of these. An exclusion that works for one form works for all. Conversely, an audit reviewer trying to "evade" your exclusion with double-encoding will fail because the WAF inspects the post-decoded form.
- **CDN caches a `403` for the asset URL.** After enforcing the exclusion, purge the cached path or the browser will keep serving the cached error.
- **GraphQL / OData navigation operators that look like traversal.** A new graph schema introducing `..` as a parent-traversal operator can trip the rule across the entire API. Add a path-scoped exclusion for the GraphQL endpoint specifically; do not exclude the whole site.
